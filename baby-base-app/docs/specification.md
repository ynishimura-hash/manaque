# Baby Base プロジェクト仕様書 (v1.0)

## 1. コンセプト
「愛媛で頑張るすべてのママを、専門家のネットワークでサポートする」をミッションとした、地域密着型の育児情報ポータルサイト。
既存の「Ehime Base」の堅実なマッチングインフラを活用しつつ、「ママスタ」のような親しみやすく、情報密度の高いUI/UXを提供。

## 2. ターゲットユーザー
- 愛媛県内に居住する妊婦・母親
- 育児の悩みを専門家に相談したいユーザー
- 地元のイベントや育児情報を効率よく収集したいユーザー

## 3. 主要機能
### 3.1 統合ポータル (Top Page)
- **AIお悩み案内所 (インラインチャット)**: 
  トップページで直接悩みを入力すると、AIが適切な専門家やコンテンツを提案。
- **専門家タイムライン (SNS)**: 
  専門家による最新の投稿（Instagram風）を表示。現場の「生の声」をリアルタイムで閲覧可能。
- **コンテンツスライダー**: 
  最新の知恵袋・コラムを横スクロールで視認性よく配置。
- **カテゴリ検索**: 
  「妊娠・出産」「離乳食」などのタイル型ボタンから、関連する専門家・記事・投稿を網羅的に検索可能。

### 3.2 スペシャリスト・マッチング
- **専門家一覧・詳細**: 
  愛媛県内の助産師、栄養士、ねんねコンサルタント等のプロフィール閲覧と相談予約。
- **SNS連携**: 
  専門家の個別ページに、その専門家の最新投稿一覧を表示。

### 3.3 ナレッジハブ (Learning)
- **知恵袋・記事**: 
  専門家による執筆記事の閲覧。
- **E-ラーニング**: 
  動画等を用いた育児スキルの習得。

### 3.4 ユーザー・オンボーディング
- **ママ登録フロー**: 
  子供の月齢、居住地（愛媛県内の市町村）、興味関心を登録し、パーソナライズされた情報を出し分け。

## 4. 技術仕様
### 4.1 アーキテクチャ
- **Ehime Baseとの分離**: 
  `LayoutWrapper`において `/babybase` パスのレイアウトを条件付きで除外。Ehime Baseのサイドバーや共通ヘッダーを表示せず、独自のナビゲーション（ピンクを基調としたデザイン）を実現。
- **共通認証システム**: 
  Ehime Baseとアカウント情報を共有しつつ、Baby Base独自のプロフィール（`momProfile`）を拡張。

### 4.2 データ構造 (Zustand / appStore)
- `bbSpecialists`: 専門家マスターデータ
- `bbEvents`: イベント情報
- `bbArticles`: コラム・知恵袋記事
- `bbPosts`: 専門家によるSNS風投稿
- `momProfile`: ママ・子供の属性情報

## 5. ディレクトリ構成 (整理後)
- `src/app/babybase/`: ルーティング・ページコンポーネント
- `src/lib/types/babybase.ts`: 型定義
- `src/lib/babybaseData.ts`: ダミーデータ・マスター
- `baby-base-app/`: プロジェクト管理・仕様書・資産（本ドキュメント含む）
